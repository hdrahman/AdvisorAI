Technology Stack Options

Three approaches for building AdvisorAI with different trade-offs

A

Vite + React + Tailwind CSS (Recommended)

Stack: Vite, React 18, React Router, Tailwind CSS, TypeScript

Pros

  • Lightning-fast dev server and HMR
  • Modern, clean component architecture
  • Tailwind matches your Figma design system perfectly
  • Small bundle size, production-ready
  • Easy to add real backend later
  • TypeScript for type safety with mock data

Cons

  • Requires learning React hooks if unfamiliar
  • More initial setup than plain HTML
B

Next.js 14 + App Router

Stack: Next.js 14, React Server Components, Tailwind CSS, TypeScript

Pros

  • File-based routing built-in
  • Server components for future backend integration
  • Great for SEO if needed later
  • Can add API routes easily
  • Modern React features

Cons

  • Heavier framework for an MVP
  • Overkill for static mock data
  • Steeper learning curve
  • Slower dev experience than Vite
C

Vanilla HTML/CSS/JS + Build Tool

Stack: HTML, CSS (or Tailwind CDN), Vanilla JavaScript/TypeScript, Vite for bundling

Pros

  • Simplest approach, no framework
  • Fast to prototype
  • Minimal dependencies

Cons

  • Repetitive code for 5 screens
  • Hard to maintain consistency
  • Manual DOM manipulation gets messy
  • Difficult to scale or refactor later
  • State management becomes painful

My Recommendation: Option A (Vite + React + Tailwind)

Given your Figma design's modern dark theme and the need for 5 interactive screens with shared navigation, Vite + React + Tailwind is the sweet spot. It's fast to develop, matches your design system, keeps components reusable, and won't feel like overkill. The component architecture will make it easy to maintain consistency across all screens, and when you're ready to add a real backend, it's straightforward to integrate APIs.
